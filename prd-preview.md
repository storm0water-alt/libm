请基于已有的档案管理系统高保真前端UI，为我增量开发档案PDF预览功能，具体要求如下：

### 1. 核心功能逻辑
- 触发方式：在档案管理列表页中，点击任意一条档案记录（整行/预览按钮均可，优先整行点击），触发PDF预览弹窗
- 文件关联：通过该条档案记录的“档案号”（唯一ID）匹配服务器中对应的PDF文件（档案号=PDF文件名）
- 预览能力：完整展示PDF文件内容，支持基础的预览操作（放大、缩小、翻页、下载、关闭）

### 2. 交互与UI要求（兼容现有高保真UI风格）
#### （1）列表页适配
- 视觉提示：档案列表中可预览的记录行，hover时鼠标变为“手型”，并在行尾增加极简的“预览”小图标（PDF标识）
- 点击反馈：点击记录行后，先显示加载状态（如弹窗内转圈），再渲染PDF内容，无卡顿感

#### （2）PDF预览弹窗
- 样式：弹窗宽度占屏幕80%、高度占屏幕75%，顶部为弹窗标题（显示“档案预览：[档案号]-[文件名]”），底部为操作按钮，中间为PDF预览区域
- 核心操作按钮：
  - 缩放控制：放大（+）、缩小（-）、重置（100%）按钮
  - 翻页控制：上一页、下一页、页码跳转（显示“第X页/共Y页”）
  - 其他：下载PDF按钮（直接下载对应文件）、关闭弹窗按钮
- 异常处理：
  - 无对应PDF文件时，弹窗内显示“暂无可用的PDF预览文件”
  - PDF加载失败时，显示“文件加载失败，请重试”，并提供“重新加载”按钮

### 3. 技术实现要求
- 兼容现有技术栈：请默认按Vue3 + Element Plus（若为React + Ant Design可备注修改）开发，使用成熟的PDF预览库（如pdfjs-dist）
- 无样式冲突：新增样式需独立命名空间，完全适配现有高保真UI的配色、间距、按钮风格
- 代码交付：
  1. 新增的PDF预览组件完整代码（含引入pdfjs-dist的配置）
  2. 档案列表页的修改代码（仅增量改动，保留原有逻辑）
  3. 清晰的注释（标注PDF文件关联逻辑、预览核心方法、异常处理）
  4. 集成说明（如依赖安装命令、组件引入方式）

### 4. 额外要求
- PDF预览为本地/服务器文件预览，无需在线解析
- 弹窗支持ESC键关闭，点击弹窗遮罩层不关闭（防止误操作）
- 预览过程不阻塞页面其他操作
