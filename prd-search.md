请基于已有的档案管理系统高保真前端UI，为我开发新增的类Google风格全局搜索功能，具体要求如下：

### 1. 核心业务流程
- 登录后跳转：用户完成登录操作后，默认跳转到极简的搜索页面（而非原列表页）
- 搜索触发：在搜索框输入关键词后，点击搜索按钮/按回车键，执行全局全文检索（非字段定向查询）
- 结果展示：检索完成后自动跳转到档案管理列表页面，列表中仅展示与关键词匹配的档案记录

### 2. 页面与交互要求
#### （1）搜索页面（登录后默认页）
- UI风格：严格对齐Google搜索页极简风格
  - 布局：页面居中，顶部仅保留系统logo/名称，核心区域为居中的大尺寸搜索框+下方2个极简按钮（“档案搜索”+“高级搜索”）
  - 搜索框：宽度默认600px（可拉伸），无预设提示文字，点击无边框变化，输入时无字段筛选下拉（突出全局搜索属性）
  - 按钮：“档案搜索”为主要按钮（浅灰底色，hover变深），“高级搜索”为次要按钮（无底色，仅文字），均无多余图标
- 交互：
  - 输入关键词后回车/点击“档案搜索”，按钮显示加载状态（转圈动画），无其他页面跳转 Loading
  - 无关键词输入时，搜索按钮置灰不可点击
  - “高级搜索”点击后仅提示“全局搜索已覆盖所有档案内容，无需字段筛选”（因核心是全局检索而非字段查询）

#### （2）检索逻辑要求
- 检索类型：全局全文检索（非字段查询），检索范围覆盖每个档案的所有元数据字段（档案号、文件名、描述、建档信息等）及PDF文件正文内容（前端仅需传递关键词，无需处理PDF解析，由后端完成全文检索）
- 结果处理：检索完成后，将匹配的档案ID/关键词等参数通过URL拼接（如?searchKey=xxx）传递到档案管理列表页

#### （3）档案列表页适配
- 接收参数：列表页解析URL中的搜索关键词参数
- 结果展示：仅展示后端返回的匹配档案记录，列表保留原有UI样式（无需修改），顶部增加“搜索结果：共X条 关键词：「xxx」”的文字提示
- 重置功能：列表页增加“返回搜索页”按钮，点击可回到登录后的默认搜索页面

### 3. 技术实现要求
- 兼容现有系统：代码需适配现有前端技术栈（请默认按Vue3 + Element Plus开发，若为React可备注修改）
- 无样式冲突：新增样式需独立命名空间，不影响原有高保真UI的布局和交互
- 代码交付：
  1. 新增搜索页面的完整代码（含HTML/CSS/JS）
  2. 登录页跳转逻辑修改代码（指向新搜索页）
  3. 档案列表页接收搜索参数、展示结果的修改代码
  4. 核心代码添加注释，说明修改点和检索参数传递逻辑

### 4. 额外要求
- 全局检索仅需前端完成参数传递和页面跳转，无需实现检索算法（由后端处理）
- 交互流畅：页面跳转无卡顿，加载状态提示清晰，无关键词时禁止触发搜索
